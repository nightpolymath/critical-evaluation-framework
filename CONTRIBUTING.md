# Contributing to the Critical Evaluation Framework

Thank you for your interest in improving the CEF. Contributions are welcome and valued.

## How to Contribute

### Reporting Issues

Found a problem? Please open a GitHub issue with:
- **Clear description** of the issue (ambiguity, error, missing case)
- **Specific lens or section** affected
- **Example or context** if applicable
- **Suggested fix** (if you have one)

### Suggesting Improvements

Have an idea for a new lens, failure mode, or refinement?

1. Open a GitHub issue describing your suggestion
2. Explain the **gap** the improvement addresses
3. Provide **concrete examples** from your use
4. Note the **intellectual foundations** if applicable

### Submitting Changes

For structural or content changes:

1. **Fork the repository**
2. **Create a branch** with a descriptive name (e.g., `improve/lens-4-causal-structure`)
3. **Make your changes**:
   - Keep edits clear and focused
   - Maintain the framework's tone and structure
   - Update the Changelog with your contribution
4. **Submit a pull request** with:
   - Clear description of changes
   - Rationale (why this improvement?)
   - Any trade-offs or limitations you considered

## Standards for Contributions

### Content

- **Clarity**: Write for practitioners, not academics. Avoid jargon unless essential.
- **Specificity**: Use examples and diagnostics. Abstract principles alone aren't actionable.
- **Honesty**: Acknowledge limitations and failure modes, don't hide them.
- **Auditability**: Changes should be traceable and justified.

### Scope

Contributions should:
- Improve **clarity, completeness, or practical utility** of the framework
- Address **genuine gaps** in the lens structure or guidance
- Be grounded in **real analytical experience** (not theoretical speculation)

Contributions should not:
- Reduce the framework to a single "best" lens (diversity of lenses is intentional)
- Remove nuance or caveats for simplicity
- Optimize for defensibility over intellectual honesty

### Attribution

All contributors will be credited in:
- The Changelog (for substantive contributions)
- The repository contributors list (GitHub auto-tracks this)

---

## Intellectual Governance

The CEF is designed around specific principles:
- **Auditability**: Explicit, traceable reasoning
- **Causal clarity**: Rigorous treatment of cause and effect
- **Epistemic discipline**: Process matters as much as output
- **Intellectual authenticity**: Genuine engagement over polished consensus

Contributions should reinforce (not undermine) these principles.
